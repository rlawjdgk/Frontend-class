3) inline-block요소 속성

*Semantic 태그
Semantic: 의미를 갖고 있는 / 의미론적인
header: 문서상에 헤더의 역할 => 머릿글 || 인트로 역할
nav: GNB || LNB 와 같은 네비게이션 목록 역할
aside: 사이드 측면 영역의 배너 혹은 지면 역할
section: 실제 문서상의 본문 역할
article: section 태그 안에 소단락(*세부영역) 나누는 역할
footer: 문서상의 푸터의 역할 -. 꼬리말 || 에필로그 역할

3가지 방법
1) 인라인 스타일

2) 내부 스타일

3) 외부 스타일(*실무)




*css3 선택자
1) 전체선택자
- wildcard 선택자
- reset css
- * == all

2) 태그선택자
- 특정한 태그를 선택해서 스타일 적용

3) 아이디선택자
- html 태그 중에서 id 속성 값을 가지고 있는 태그들에게
스타일을 적용
- "#"
- id 속성값은 반드시 html에서 1번만 사용해야

4) 클래스선택자
-html 태그 중에서 class 속성 값을 가지고 태그들에게 스타일을 적용
- "."
- class 속성 값은 html에서 여러번 횟수제한없이 사용가능
- 반복적으로 공통된 스타일 정의하고 적용

5) 후손선택자
- 부모 아래에 들어오는 모든 자식요소들
- 자손선택자도 모두 포함
- "띄어쓰기"

6) 자손선택자 
- 부모의 직계 자식요소
- ">"

7) 속성선택자
- 동일한 html 태그의 동일한 속성을 가지고 있는
경우, 안에 들어다있는 속성값에 따라서 스타일 상이하게 적용

8) 동위선택자
- 동: 같을 "동"
- 위: "위"치
> 같은 위치에 속해있는 욕소들 간 스타일을 적용하고 싶을 때 => 반드시 형제 요소
> 특정 요소의 바로 뒤에 존재하는 형제 : "+"
> 특정 요소 뒤에 있는 모든 형제 : "~"

9) 구조선택자
- 일반구조선택자
- 속성구조선택자

10) 반응선택자
- 사용자가 어떤 액션(*이벤트) 실행 => 반응을 일으킴 (마우스 오버 / 클릭 / 떠나게)
- hover: 마우스 오버
- active: 마우스 클릭

11) 상태선택자
- 사용자의 이벤트 액션을 통해서 특정 태그 및 요소들의 상태가 변화되었다는 것을 감지하고 이를 통해서 스타일 적용
- form: 버튼 / check / radio

12) 링크선택자

13) 문자선택자

14)부정선택자
